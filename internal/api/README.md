# api 目录

## 目录说明
API接口层，负责HTTP请求处理、路由定义和中间件管理。

## 功能描述
- 处理HTTP请求和响应
- 定义RESTful API路由
- 实现认证、授权、限流等中间件
- 参数验证和错误处理

## 目录结构
```
api/
├── handlers/      # 请求处理器
├── middleware/    # 中间件
└── routes/        # 路由定义
```

## 职责说明
- **handlers**: 处理具体的HTTP请求，调用service层
- **middleware**: 实现跨切面功能（认证、日志、限流等）
- **routes**: 定义路由映射和API版本管理

## 开发规范
- 处理器只负责请求解析和响应格式化
- 业务逻辑委托给service层处理
- 统一的错误处理和响应格式
